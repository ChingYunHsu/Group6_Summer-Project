# `venues_export.sql` schema contract (2026-07-18)

`venues_export.sql` was generated by commit
`ae63fbb50d7b963b4b418ba24959d09bed59acef`, titled
`7.18 fixed bug in DB deplyment`. It is a MySQL data-only dump and its
`INSERT INTO venues VALUES (...)` statements contain **31 positional values**.

Do not import those statements into a schema with a different ordinal layout.
Convert them to an `INSERT INTO venues (<columns...>) VALUES (...)` import, or
create a fresh schema from this repository (which now preserves this order).

The base definition is
`docker/mysql/init/001_clearpath_schema.sql` at commit `ae63fbb`. Columns
25–31 had been added dynamically by the V2 label writeback workflow, and are
therefore documented explicitly here. In the current schema they are declared
at the end of `venues` in the same order; migration
`012_add_venues_prediction_label_columns.sql` adds them to existing databases.

`accessible_status` in that export used the then-valid values
`full_access`, `partial`, `step_free_route_only`, and `none`. The current enum
adds `unknown`; it is a backward-compatible superset and does not change the
export field order.
