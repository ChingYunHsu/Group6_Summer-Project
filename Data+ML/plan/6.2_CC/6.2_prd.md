# ClearPath Database PRD — 2026.6.2

## 1. 项目背景

ClearPath 是一款面向曼哈顿弱势游客的 Waze 风格应用，提供实时无障碍警报、拥挤度报告和 ML 预测。数据库是整个系统的数据基础，支撑 12 个 REST API 端点和 4 大功能柱（Find, Report, Predict, Assist）。

| 文档 | 核心约束 |

### 3.3 非功能需求

| 需求 | 指标 |
|------|------|
| 查询延迟 | 地图 POI 查询 < 200ms（1000 条记录内） |
| 空间查询 | 支持 lat/lng 范围查询（曼哈顿 bounding box） |
| 并发 | 支持 50+ 同时在线用户（演示级别） |
| 数据一致性 | 导入时去重（source + source_id 唯一） |
| 字符集 | UTF-8mb4（支持多语言） |
| 过期清理 | reports 2 小时自动过期；cache 表按 expires_at 清理 |

## 5. 技术约束

- **数据库**: MySQL 8.0+（Docker 部署）
- **字符集**: utf8mb4
- **引擎**: InnoDB（支持事务和外键）
- **空间索引**: 使用 DECIMAL(10,7) 存储 lat/lng + 复合索引（MySQL 8.0 对 DECIMAL 的空间索引支持有限，备选方案用 POINT 类型）
- **ORM**: Flask-SQLAlchemy 或原生 PyMySQL
- **迁移**: 手动 SQL 脚本（docker-entrypoint-initdb.d）

## 6. 验收标准

| # | 标准 | 验证方式 |
|---|------|---------|
| AC1 | 所有 9 张表创建成功，无报错 | `SHOW TABLES` + `DESCRIBE` 每张表 |
| AC2 | 外键约束正确 | 尝试插入无效外键，应报错 |
| AC3 | 导入脚本可运行 | 执行 ETL 脚本后 `SELECT COUNT(*)` > 0 |
| AC4 | 空间查询可用 | `SELECT * FROM venues WHERE lat BETWEEN ... AND lng BETWEEN ...` |
| AC5 | 去重机制生效 | 重复导入同一数据源，记录数不增加 |
| AC6 | API 端点可查询 | 每个端点返回 200 + 非空数据 |
| AC7 | Docker 启动自动建表 | `docker-compose up` 后表自动存在 |
