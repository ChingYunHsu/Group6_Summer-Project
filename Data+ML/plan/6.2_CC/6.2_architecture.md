## 4. 索引策略

| 表 | 索引 | 类型 | 用途 |
|---|------|------|------|
| `venues` | `(venue_type, borough)` | 复合 | 按类型+区域过滤 |
| `venues` | `(lat, lng)` | 复合 | 空间范围查询 |
| `venues` | `(source, source_id)` | UNIQUE | 导入去重 |
| `toilets` | `(borough, is_operational)` | 复合 | 按区域+状态过滤 |
| `toilets` | `(lat, lng)` | 复合 | 空间范围查询 |
| `reports` | `(status, expires_at)` | 复合 | 过期清理 + 活跃查询 |
| `reports` | `(venue_id, status)` | 复合 | 某地点的活跃报告 |
| `busyness_predictions` | `(venue_id, prediction_date, hour)` | 复合 | 核心查询路径 |
| `accessibility_infra` | `(borough, condition)` | 复合 | 按区域+状态过滤 |
| `traffic_cache` | `(expires_at)` | 普通 | 过期清理 |
| `weather_cache` | `(observation_time)` | 普通 | 时间范围查询 |

## 5. Docker 部署结构

```
docker/
└── mysql/
    └── init/
        ├── 001_clearpath_schema.sql    ← 建表 DDL
        ├── 002_seed_venues.sql         ← 导入 OSM + NYS + AED
        ├── 003_seed_toilets.sql        ← 导入两个厕所源
        ├── 004_seed_accessibility.sql  ← 导入 Ramps
        └── 005_seed_indexes.sql        ← 补充索引（如需）
```

Docker 启动时自动执行 `docker-entrypoint-initdb.d/` 下的 SQL 文件（按文件名排序），实现一键建表+导入。

## 6. ETL 流程

```
数据源文件 (CSV/GeoJSON)
       │
       ▼
┌──────────────────┐
│  ETL Python 脚本  │  ml_training/etl/
│  - 读取原始文件    │
│  - 字段标准化     │
│  - 去重检查       │
│  - 哈希计算       │
└────────┬─────────┘
         │
         ▼
┌──────────────────┐
│   MySQL INSERT   │  INSERT ... ON DUPLICATE KEY UPDATE
│   或 LOAD DATA   │
└──────────────────┘
```

### ETL 脚本目录结构

```
ml_training/etl/
├── config.py           ← 数据库连接配置
├── utils.py            ← 通用工具（哈希、坐标转换）
├── etl_venues.py       ← 导入 venues (OSM + NYS + AED)
├── etl_toilets.py      ← 导入 toilets (NYC + Parks)
├── etl_accessibility.py← 导入 ramps
├── etl_weather.py      ← 缓存天气数据
├── etl_traffic.py      ← 缓存交通数据
└── run_all.py          ← 一键执行全部 ETL
```
