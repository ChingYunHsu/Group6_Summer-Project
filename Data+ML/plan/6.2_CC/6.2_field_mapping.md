# ClearPath 数据源字段映射 — 2026.6.2

## 1. venues 表（Healthcare + AED 合并）

### 1.1 OSM Healthcare POI → venues

| 源字段 (GeoJSON properties) | 目标字段 | 转换逻辑 |
|---------------------------|---------|---------|
| `@id` | `source_id` | 直接映射，格式如 `node/123456` |
| `name` | `name` | 直接映射 |
| `healthcare` / `amenity` | `venue_type` | 映射: `clinic`→`clinic`, `hospital`→`hospital`, `pharmacy`→`pharmacy`, `dentist`→`dentist`, `doctors`→`clinic` |
| `geometry.coordinates[1]` | `lat` | 纬度 |
| `geometry.coordinates[0]` | `lng` | 经度 |
| `addr:street` + `addr:housenumber` | `address` | 拼接: `{housenumber} {street}` |
| `addr:city` | `city` | 直接映射，默认 `Manhattan` |
| `addr:postcode` | `zip_code` | 直接映射 |
| — | `source` | 固定值 `'osm'` |
| — | `data_confidence` | 固定值 `0.6`（OSM 社区数据） |
| — | `borough` | 需通过 lat/lng 反查或留 NULL |

**去重规则**: `source = 'osm'` + `source_id = @id`

### 1.2 NYS Health Facility → venues

| 源字段 (CSV) | 目标字段 | 转换逻辑 |
|-------------|---------|---------|
| `Facility ID` | `source_id` | 直接映射 |

| `Description` | `venue_type` | 映射: `Hospital`→`hospital`, `Diagnostic and Treatment Center`→`clinic`, `Residential Health Care`→`clinic`, 其他→`clinic` |
| `Facility Latitude` | `lat` | 直接映射（注意部分为空） |
| `Facility Longitude` | `lng` | 直接映射（注意部分为空） |
| `Facility Address 1` + `Facility Address 2` | `address` | 拼接 |
| `Facility City` | `city` | 直接映射 |
| `Facility Zip Code` | `zip_code` | 直接映射 |
| `Facility Phone Number` | `phone` | 直接映射，格式化为 `xxx-xxx-xxxx` |
| `Facility Website` | `website` | 直接映射 |
| `Facility County` | `borough` | 映射: `New York`→`Manhattan`, `Kings`→`Brooklyn`, `Queens`→`Queens`, `Bronx`→`Bronx`, `Richmond`→`Staten Island` |
| `Short Description` | `facility_type` | 直接映射 |
| `Operator Name` | `operator_name` | 直接映射 |
| `Ownership Type` | `ownership_type` | 直接映射 |
| — | `source` | 固定值 `'nys_health'` |
| — | `data_confidence` | 固定值 `0.9`（官方数据） |

**去重规则**: `source = 'nys_health'` + `source_id = Facility ID`
**过滤条件**: 仅导入 `Facility County` 在 NYC 五区的记录

| 源字段 (CSV) | 目标字段 | 转换逻辑 |
|-------------|---------|---------|

| `Longitude` | `lng` | 直接映射 |

| `Hours of Operation` | `hours_of_operation` | 直接映射 |
| `Operator` | `operator` | 直接映射 |
| `Additional Notes` | `additional_notes` | 直接映射 |
| `Website` | `website` | 直接映射 |
| — | `source` | 固定值 `'nyc_restrooms'` |
| — | `source_id` | 生成: 哈希(Facility Name + Latitude + Longitude) |

**去重规则**: `source = 'nyc_restrooms'` + `source_id` = 哈希

### 2.2 Parks Toilets → toilets

| 源字段 (CSV) | 目标字段 | 转换逻辑 |
|-------------|---------|---------|
| `Name` | `name` | 直接映射 |
| `Location` | `address` | 直接映射（注意：无坐标，需地理编码） |
| — | `lat` | 需通过 `Location` + `Borough` 地理编码 |
| — | `lng` | 需通过 `Location` + `Borough` 地理编码 |
| `Borough` | `borough` | 直接映射 |
| `Handicap Accessible` | `accessibility` | 映射: `Yes`→`accessible`, 空→`unknown` |
| `Open Year-Round` | `open_year_round` | 映射: `Yes`→`TRUE`, `No`→`FALSE` |
| `Comments` | `additional_notes` | 直接映射 |
| — | `source` | 固定值 `'parks_toilets'` |
| — | `source_id` | 生成: 哈希(Name + Borough) |
| — | `is_operational` | 默认 `TRUE`（公园厕所一般开放） |
| — | `operator` | 固定值 `'NYC Parks'` |

**注意**: Parks Toilets 没有经纬度，需要通过地理编码服务获取坐标，或在 ETL 中使用 geocoding API。

## 4. 缓存表

| API 响应字段 | 目标字段 | 转换逻辑 |
|-------------|---------|---------|
| 请求参数 | `origin_lat`, `origin_lng` | 用户当前位置 |

| 请求参数 | `travel_mode` | walking/driving/transit |
| `duration_in_traffic.value` | `duration_seconds` | 直接映射 |
| `distance.value` | `distance_meters` | 直接映射 |
| — | `traffic_level` | 根据 duration/distance 比率计算 |

| — | `fetched_at` | 当前时间 |
| — | `expires_at` | 当前时间 + 30 分钟 |

## 5. 数据质量注意事项

| 问题 | 数据源 | 处理方案 |
|------|--------|---------|
| NYS Health 部分记录无坐标 | NYS Health Facility | 跳过无坐标记录，记录日志 |
| Parks Toilets 无坐标 | Parks Toilets | 需地理编码，或暂时跳过 |
| OSM 数据可能超出 NYC 范围 | OSM Healthcare | 过滤 lat 40.4-40.9, lng -74.3--73.7 |
| AED 无标准化 ID | AED Inventory | 用 Entity_Name + Address 生成哈希 |
| Borough 编码为数字 | Pedestrian Ramps | 1-5 映射为区名 |
| 部分字段为空 | 所有 | 设置 DEFAULT 值，不阻塞导入 |
